## Q&A Agent Template

Position: Answers questions automatically using your existing knowledge.

### Structure
1. Capability (what it does)
2. Value (eliminates repetitive questions)
3. Proof (metrics + quote)

### Template
```
[Agent Name] answers questions so your team doesn't have to.

What it does: Pulls answers from [your knowledge base / docs / connected tools] 
automatically — no human needed to find or format the response.

The result: [X] questions answered per [day/week] by the agent. Your team 
gets instant answers; you get your time back.

"[Quote about eliminating repetitive questions]" — [Name], [Role], [Company]
```

### Example
```
Product Oracle answers dozens of questions every day about the 
product roadmap and how features work — questions that previously 
required a product manager to respond manually.

"We used to monitor those channels constantly. Now our agents 
answer questions automatically, and we just audit responses and 
feed improvements back into the Custom Agent." — Ben Levick, 
Head of Operations and Internal AI, Ramp
```

## Task Routing Agent Template

Position: Auto-categorizes and routes incoming work to the right people.

### Structure
1. The problem (manual triage pain)
2. The solution (automatic routing)
3. The outcome (time saved + accuracy)

### Template
```
Incoming [requests/tickets/emails] → routed to the right person → done.

What it does: Watches for new items in [source], categorizes by [criteria], 
enriches with [context], and routes to the right owner with all details attached.

The result: [X] items routed per [day/week]. [X]% auto-resolved without human 
intervention. No more manual copy-pasting between tools.

"[Quote about eliminating triage work]" — [Name], [Role], [Company]
```

### Example
```
Agent watches for new requests in [helpdesk], automatically categorizes 
by [issue type], enriches with [customer context], creates a task in 
[Linear], and notifies the right owner. Done in seconds.

Real result: Remote's IT team saved 20 hours/week with 95%+ triage 
accuracy. 25%+ of tickets resolved autonomously.
```

## Status Update Agent Template

Position: Writes recurring reports so you don't have to.

### Structure
1. The pain (manual reporting time)
2. The solution (automated summarization)
3. The outcome (time freed)

### Template
```
Weekly reports written themselves. Delivered on your schedule.

What it does: Queries [databases/sources] for [status = complete], groups by 
[assignee/category], and writes a formatted summary to [destination] — ready 
for your review before sending.

The result: [X] minutes per report → [Y] seconds. Your [Monday/Friday] updates 
write themselves.
```

### Example
```
Every Friday at 4pm, the agent queries the [database] for tasks 
with [Status = Done] and [Due Date = this week], groups by 
[Assignee], and posts a formatted summary to #team-updates.

Result: What took 30 minutes now takes 3. Your Friday updates 
write themselves.
```

## Custom Agent (Build Your Own) Template

Position: Describe what you need, AI builds it — no code required.

### Structure
1. The capability (natural language setup)
2. The scope (if you can describe it, it can do it)
3. Social proof (what others have built)

### Template
```
Describe what you need. [Product] AI builds it.

What it does: Tell your agent what recurring task you do manually — in plain 
language. It creates the workflow, sets up triggers, and starts running. No 
code required.

The result: If you can describe it, your agent can do it. [X] specific examples 
of custom agents teams have built in the first [weeks/months].
```

### Example
```
One word to describe Custom agents: crazy good.

Anything you can do in Notion, your Notion Agent can do for you.

In the first few weeks, early users built over 21,000 Custom Agents. 
Notion itself runs 2,800 — more agents than employees.
```
